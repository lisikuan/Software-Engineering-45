## 关键设计决策

### 1. 无数据库设计
- ✅ 使用 JSON 文件作为数据存储
- ✅ 每个实体类型一个 JSON 文件
- ✅ 使用临时文件方式避免部分写入

### 2. 分层架构
- Presentation (UI) → 提交用户输入
- Controller → 路由和基本验证
- Service → 业务逻辑
- Repository (DAO) → 数据访问
- JSON Storage → 持久化

### 3. 异常处理
- DataAccessException: 文件I/O失败
- ValidationException: 输入验证失败
- BusinessException: 业务规则违反

### 4. ID 生成
- User ID: "user_" + UUID 前8位
- 其他实体类似方式

### 5. 时间戳
- LocalDateTime 自动序列化为 ISO 8601 格式
- 所有创建操作自动设置创建时间

## 遵循 AGENTS.md 规范情况

✅ 层级隔离 - 各层职责清晰
✅ 异常处理 - 使用自定义异常
✅ JSON 持久化 - 单一真实来源
✅ 枚举使用 - 状态值使用枚举而非字符串
✅ 验证逻辑 - 在 Service/Validation 层
✅ 约束遵守 - 无数据库、无 Spring
